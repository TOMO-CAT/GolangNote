## 编写原则

#### 1. 二八原则

遵循“二八原则”，记录**Golang后端开发**中百分之八十以上的知识点，避开各类奇技淫巧。

#### 2. 删繁就简

对于强记忆性的内容简单介绍，比如枯燥的进制转换和少用的语言特性等。
